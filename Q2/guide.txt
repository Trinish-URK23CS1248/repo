Experiment: Creating a Compute Engine VM without External IP & Enabling Private Google Access
🎯 Aim

To create a Compute Engine Virtual Machine without an external IP address and enable Private Google Access on the subnet to securely access Google services without using the public internet.

📘 Description

In Google Cloud, VM instances usually get an external IP for internet access. But for security-focused architectures, we avoid exposing VMs publicly.

No External IP → VM is not accessible from the internet
Private Google Access → Allows VM to access Google services (like Cloud Storage, APIs) internally
Ensures:
Better security 🔒
Controlled communication
Reduced attack surface

This setup is commonly used in:

Enterprise systems
Backend services
Secure DevOps pipelines

⚙️ Procedure
Step 1: Open Google Cloud Console
Go to: https://console.cloud.google.com
Sign in with lab credentials
Select your project
🔹 Step 2: Create Custom VPC Network
Go to Navigation Menu (☰) → VPC Network → VPC Networks
Click Create VPC Network

Fill details:

Name: private-vpc
Subnet creation mode: Select Custom
🔹 Step 3: Create Subnet & Enable Private Google Access
Under New Subnet, enter:
Name: private-subnet
Region: us-central1 (or given region)
IP range: 10.10.0.0/24
Scroll down and enable:
✅ Private Google Access
Click Done
Click Create
🔹 Step 4: Create Firewall Rule (for SSH via IAP)
Go to VPC Network → Firewall
Click Create Firewall Rule

Enter:

Name: allow-iap-ssh
Network: private-vpc
Direction: Ingress
Action: Allow

Under Targets:

Select All instances in the network

Under Source filter:

IPv4 ranges:
35.235.240.0/20

Under Protocols and ports:

Select Specified protocols
Tick tcp → enter 22
Click Create
🔹 Step 5: Create VM without External IP
Go to Navigation Menu → Compute Engine → VM Instances
Click Create Instance

Fill details:

Name: private-vm
Region: us-central1
Zone: us-central1-a
Machine type: e2-micro

👉 Click Networking

Network: private-vpc
Subnetwork: private-subnet

⚠️ Important:

Under External IP → Select None
Click Create
🔹 Step 6: Connect to VM using IAP

Open Cloud Shell (top right terminal icon)

Run:

gcloud compute ssh private-vm --zone=us-central1-a --tunnel-through-iap

👉 Type Y if prompted

🔹 Step 7: Verify Private Google Access

Inside VM, run:

curl https://www.googleapis.com

✔ If output appears → success
✔ Confirms access without external IP

🔹 Step 8: (Optional Check)

Check VM details:

No External IP assigned
Only Internal IP present✔ If response comes → Private Google Access is working
✔ No external IP used

🧪 Observations
VM has no external IP
Cannot be accessed directly from internet
Still able to access Google services internally
Secure communication is achieved
📊 Result

Successfully created a secure Compute Engine VM without external IP and enabled Private Google Access, allowing internal access to Google services.
